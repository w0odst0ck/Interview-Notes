# 四类主流套利模式・统计套利

## 一句话定义

**不看产业链、不看现货，只看数据规律：找两个历史走势高度相关的资产，当它们 “走势脱节” 时对冲入场，等恢复同步后平仓获利。**

也常被直接叫做：**配对交易（Pairs Trading）**

---

# 1. 核心逻辑

1. 找出 **长期走势高度相关** 的一对资产
    
    比如：A 涨 B 大多涨，A 跌 B 大多跌
2. 计算它们的 **价差 / 比值**
3. 当价差**异常偏离历史均值**时
    
    - 偏贵的 → 做空
    - 偏便宜的 → 做多
    
4. 等价差**回归正常** → 双边平仓

本质：

**用历史统计规律，赌 “不正常” 会变回 “正常”。**

---

# 2. 和另外三种套利的核心区别

- **期现套利**：现货 ↔ 期货，有**交割强制回归**
- **跨期套利**：同品种不同月，有**时间强制回归**
- **跨品种套利**：上下游 / 替代品，有**产业链逻辑**
- **统计套利**：
    
    没有硬约束回归，
    
    回归动力 = **历史规律 + 资金行为**

所以它**更依赖量化模型**，也更适合你这种 AI / 数据背景。

---

# 3. 做统计套利的两个核心前提

## （1）高相关

相关系数通常要求 **> 0.8**

不然就是乱配对。

## （2）协整（Cointegration）

比相关更重要：

- 虽然短期会偏离
- 但长期会 “拽回来”
- 价差是**均值回归**的

只有协整，策略才不会长期发散亏到底。

---

# 4. 标准策略流程（非常经典）

1. 选一对股票 / ETF / 期货：如 A & B
2. 计算价差：`Spread = A - k*B`
3. 算价差的 **均值 μ** 和 **标准差 σ**
4. 计算 **Z-score** 看偏离程度
5. 入场：
    
    - Z 过高 → 空 A、多 B
    - Z 过低 → 多 A、空 B
    
6. 回归到均值附近 → 平仓

这就是机构最常用的 **Z-score 配对交易**。

---

# 5. 典型配对例子

- 工商银行 ↔ 建设银行
- 茅台 ↔ 五粮液
- SPY（标普 500） ↔ QQQ（纳指 100）
- 日经 225 ETF ↔ 东证 TOPIX ETF
- 同一行业两只龙头股

---

# 6. 优点

- 不看大盘牛熊，**市场涨跌都能做**
- 双边对冲，回撤小、曲线稳
- 机会多，遍地都能找配对
- 非常适合 **Python 量化、机器学习**

# 7. 风险（关键）

- **历史规律失效**：相关性 / 协整破裂
- 价差长期不回归，**死扛大亏**
- 黑天鹅时一起暴跌，对冲失效
- 容易过度拟合，回测很美实盘拉胯

---

# 8. 极简总结

**统计套利 = 数据驱动的配对交易**

**高相关 + 协整 = 能做**

**Z-score 偏离 = 入场**

**均值回归 = 赚钱**

**规律一破 = 风险**
