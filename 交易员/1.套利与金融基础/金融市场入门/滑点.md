# 滑点（Slippage）

用**套利 / 量化 / 写代码最实用的方式**一次性讲透，不搞玄学。

---

# 1. 滑点是什么（一句话）

**你下单时看到的价格，和最终实际成交的价格之间的差额，就是滑点。**

- 你看到：100 元
- 实际成交：100.1 元
    
    → 滑点 = **+0.1 元（不利滑点）**

滑点本质是：

**行情在你下单到成交的这段时间里，动了。**

---

# 2. 滑点为什么会产生（工科视角）

三个核心原因：

1. **网络延迟 + 撮合延迟**
    
    你发单 → 券商 → 交易所 → 撮合
    
    这期间价格已经变了。
    
2. **盘口流动性不足**
    
    你要买 100 手，但卖一只有 20 手，
    
    剩下 80 手只能吃卖二、卖三……
    
    → 价格越吃越高。
    
3. **剧烈波动（跳空、秒杀、大单冲击）**
    
    财报、突发新闻、大盘暴跌时，价格一秒跳好几档。
    

---

# 3. 滑点的两种类型

### 正向滑点（赚了，极少）

- 你挂 100 买，结果 99.9 成交
- 套利里基本不用指望

### 负向滑点（亏了，常态）

- 买高了
- 卖低了
    
    **所有套利策略都必须把它当固定成本算进去。**

---

# 4. 滑点对套利的致命影响

套利本来就是**赚几厘、几分的薄利**，

滑点稍微大一点，利润直接消失甚至亏损。

例子：

- 套利价差：0.1 元
- 滑点：0.15 元
    
    → **套利变亏损**

所以：

**滑点 > 预期价差 → 策略无效**

---

# 5. 不同市场滑点大小（实战经验值）

### 美股 ETF / 股指期货（ES、NQ）

- 滑点极小：**0.01% ~ 0.03%**
- 流动性越好，滑点越接近最小变动价位（tick）

### A 股股票 / ETF

- 正常：**0.01% ~ 0.05%**
- 涨跌停 / 小盘股：滑点可能达到 **0.5%~2%**

### 日股

- 滑点普遍比美股大
- 低价股 tick 大，滑点更明显

### 数字货币

- 滑点极大，**0.1%~2%** 都正常

---

# 6. 写策略时怎么模拟滑点（Python 直接用）

回测必须加滑点，否则收益全是假的。

通用经验值（行业标准）：

- 低流动性品种：**0.1%**
- 中流动性：**0.05%**
- 高流动性（美股 ETF、股指期货）：**0.01% ~ 0.02%**

```python
# 实战常用
SLIPPAGE = 0.0005   # 0.05% 滑点

# 买入时成本变高
real_buy_price = order_price * (1 + SLIPPAGE)

# 卖出时收入变低
real_sell_price = order_price * (1 - SLIPPAGE)
```

如果你做**高频 / 套利**，建议用更严格的：

```python
SLIPPAGE = 0.001  # 0.1% 保守回测
```

---

# 7. 滑点怎么控制（套利必看）

1. **只做高流动性品种**
    
    成交量大、盘口厚 → 滑点小
    
2. **避免市价单，用限价单**
    
    市价单 = 主动吃单 = 滑点拉满
    
    限价单 = 被动挂单 = 滑点接近 0
    
3. **拆单交易**
    
    大单拆成小单，避免吃穿盘口
    
4. **避开剧烈波动时段**
    
    开盘、收盘、财报、数据公布时滑点爆炸
    

---

# 8. 滑点 + 手续费 = 真实套利成本

真正的套利门槛：

**价差 > 手续费 + 双边滑点**

不计算滑点的套利策略 = 纸上谈兵。
