
结合你的**AI 专业背景 + Python/C++ 基础**，我为你定制了一条「从 0 到面试」的套利量化学习路径，目标是在 3-6 个月内达到可投递岗位的水平。

---

## 📅 阶段一：套利与金融基础（1 个月）

### 核心目标

搞懂套利的底层逻辑，建立金融市场认知，避免 “只会写代码不懂业务”。

### 学习内容

1. **套利理论精读**
    
    - 精读你手上的这份套利文档，重点吃透：
        
        - 三大基础原则：**相关性、对冲、均值回归**
        - 四类主流套利模式：跨品种 / 跨期 / ETF 期现 / 统计套利
        - 风控铁律：双向对冲、禁止裸头寸、单笔风险控制
        
    - 补充书籍：《量化投资策略与技术》（丁鹏）→ 套利章节
    
2. **金融市场入门**
    
    - 学习国内期货 / ETF / 股票的基本交易规则、合约规格、涨跌幅限制
    - 了解价差、基差、升贴水、滑点、手续费等核心概念
    - 推荐课程：B 站「量化交易入门」系列（免费）
    
3. **Python 金融数据处理**
    
    - 用 `Pandas` 处理历史行情数据，计算价差、相关性、协整关系
    - 用 `Matplotlib/Seaborn` 可视化价差走势，识别均值回归区间
    

---

## 📅 阶段二：统计与量化建模（1.5 个月）

### 核心目标

用 AI 专业的统计 / 机器学习能力，落地套利策略的识别与回测。

### 学习内容

1. **统计套利核心工具**
    
    - 时间序列分析：平稳性检验（ADF）、协整检验（Johansen）、误差修正模型（ECM）
    - 均值回归策略：配对交易（Pairs Trading）的数学原理与实现
    - 推荐资料：《Statistical Arbitrage》（Andrew Pole）、B 站「配对交易」实战教程
    
2. **Python 量化库实战**
    
    - 数据获取：用 `Tushare`/`AkShare` 获取免费的期货 / ETF 历史数据
    - 回测框架：学习 `Backtrader`/`VectorBT`，搭建自己的套利回测环境
    - 实战项目：实现一个**经典配对交易策略**（如黄金 - 白银、豆油 - 棕榈油）
    
3. **C++ 量化方向补强**
    
    - 重点学习：高性能计算、多线程编程、内存管理
    - 目标：理解量化交易系统中「低延迟」的实现逻辑，为后续高频 / 低延迟岗位做准备
    - 推荐资料：《Effective Modern C++》、《High-Frequency Trading》
    

---

## 📅 阶段三：策略实现与项目打磨（1.5 个月）

### 核心目标

完成 2-3 个可展示的套利项目，形成作品集，用于面试。

### 项目推荐

1. **项目 1：ETF 期现套利**
    
    - 逻辑：利用 ETF 价格与 IOPV（实时参考净值）的价差进行套利
    - 技术栈：Python + Tushare + Backtrader
    - 产出：回测报告（收益曲线、最大回撤、夏普比率）
    
2. **项目 2：统计套利（配对交易）**
    
    - 逻辑：筛选高度相关的品种（如螺纹钢 - 铁矿石），当价差偏离均值时开仓，回归时平仓
    - 技术栈：Python + Pandas + Statsmodels
    - 产出：完整的策略代码 + 回测结果 + 风控说明
    
3. **项目 3（进阶）：跨期套利**
    
    - 逻辑：利用同一商品不同月份合约的价差波动套利
    - 技术栈：Python + 期货数据 API + 滑点 / 手续费模拟
    - 产出：模拟实盘绩效报告
    

### 面试准备

- 把项目代码上传到 GitHub，写清晰的 README，突出：
    
    - 策略逻辑、回测指标、风控措施
    - 你解决的技术难点（如数据清洗、过拟合处理）
    
- 准备面试题：
    
    - 解释套利与趋势交易的区别
    - 如何判断两个品种是否适合配对交易
    - 如何控制套利中的风险（如价差不回归、流动性风险）

## 📅 阶段四：面试冲刺与岗位投递（1 个月）

### 核心目标

针对性准备面试，拿到量化 / 套利岗位 offer。

### 学习内容

1. **技术面试刷题**
    
    - Python：LeetCode 中等难度算法题（重点：数组、字符串、动态规划）
    - C++：指针、内存管理、多线程、STL 容器高频考点
    - 量化：手写配对交易核心逻辑、解释协整检验步骤
    
2. **业务面准备**
    
    - 能清晰讲述你的套利项目：策略思路 → 数据处理 → 回测 → 风控 → 结果分析
    - 了解目标公司的业务：是做高频套利、还是中低频统计套利
    
3. **简历优化**
    
    - 突出：AI 专业背景 + Python/C++ 能力 + 量化套利项目
    - 用数据说话：比如 “我的配对交易策略回测年化收益 25%，最大回撤 5%”

## 💡 结合你背景的优势

- **AI 专业**：天然具备统计、机器学习、数据分析能力，这是统计套利的核心竞争力
- **Python/C++ 基础**：可以快速跳过编程入门，直接聚焦量化策略实现
- **学习路径**：建议先从 Python 实现套利策略，再用 C++ 补强低延迟 / 系统开发能力，适配不同岗位需求
