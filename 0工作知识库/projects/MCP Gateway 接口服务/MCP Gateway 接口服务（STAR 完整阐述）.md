# 基于 STAR 法则：突出**MCP 服务**的项目架构详细阐述

（全程聚焦**MCP 服务**的设计、实现、价值，弱化通用 Web 架构，强化 MCP 核心定位）

## 一、S（Situation）场景：MCP 服务的核心定位与背景

### 项目核心场景

本项目是**AI 原生生态下的标准化 MCP（Model Control Protocol/Model Context Protocol）服务**，核心使命是**作为 AI 大模型与外部系统（数据库、文件、工具、第三方服务）的标准化桥梁**，解决 AI 模型无法安全、规范、统一调用外部能力的行业痛点。

### 关键背景

1. AI 大模型本身无直接访问外部数据 / 工具的能力，需要**标准化、轻量化的 MCP 服务**提供统一调用入口；
2. 业务核心需求：AI 通过自然语言，**安全、受控、可扩展**地查询数据库、执行数据操作；
3. 技术约束：必须严格遵循**MCP 协议规范**，支持标准 MCP 客户端 / 网关接入，同时保证服务轻量化、可移植、高兼容；
4. 架构目标：打造**独立、可插拔、可复用**的 MCP 服务，而非单纯的 Web 接口服务。

---

## 二、T（Task）任务：MCP 服务架构核心设计目标

围绕**MCP 服务**本身，架构必须完成以下核心任务：

1. **实现标准兼容的 MCP 服务端**：完整支持 MCP 协议定义的工具注册、指令调用、权限校验、响应返回规范；
2. **MCP 服务模块化解耦**：将 MCP 协议层、业务能力层、数据层完全分离，保证 MCP 服务可独立部署、独立升级；
3. **MCP 服务能力扩展**：支持动态注册新的 MCP 工具（如自然语言转 SQL、SQL 查询、文件读取等），不修改协议核心代码；
4. **MCP 服务安全与管控**：在 MCP 协议层实现指令鉴权、操作限制、日志审计；
5. **MCP 服务互联互通**：支持对接 MCP Gateway、AI 客户端（如 IDE、AI 助手），实现标准化接入。

---

## 三、A（Action）行动：MCP 服务架构核心实现（重点突出）

### 1. 整体架构：以 MCP 服务 为核心的分层设计

整个项目**以 MCP 服务为中心**，所有模块均为支撑 MCP 服务运行：

```plaintext
AI客户端 / MCP网关
        ↓
┌─────────────────────────────┐
│        MCP 协议服务层       │  ← 核心层
├───────────┬─────────────────┤
│ MCP工具层 │  MCP依赖注入层   │
├───────────┴─────────────────┤
│        数据/外部服务层       │
└─────────────────────────────┘
```

### 2. MCP 服务核心层实现（最关键）

- **MCP 协议适配**：基于标准 MCP SDK 实现完整服务端，处理`initialize`、`tools/list`、`tools/call`等标准 MCP 指令；
- **MCP 服务入口**：`app/main.py` 不只是 FastAPI 入口，**更是 MCP 服务的启动入口**，同时支持 HTTP 和 MCP 双协议；
- **MCP 路由管理**：`app/routers/mcp.py` 是**MCP 服务的核心路由**，统一注册所有 MCP 工具、处理所有 MCP 客户端请求；
- **MCP 标准化响应**：所有请求严格返回 MCP 协议规定的 JSON 结构，保证任何 MCP 客户端可直接解析。

### 3. MCP 工具层：可插拔的业务能力

MCP 服务采用**工具化架构**，所有业务功能都是 MCP 标准工具：

- `nl2sql_mcp_tool`：自然语言转 SQL（MCP 标准工具）
- `sql_executor_mcp_tool`：安全执行 SQL 查询（MCP 标准工具）
- 新增能力只需**新增 MCP 工具文件**，注册到 MCP 路由即可，无侵入式修改

### 4. MCP 服务依赖与配置

- `dependencies.py` 专为 MCP 服务提供依赖：数据库连接、大模型客户端、MCP 会话管理；
- `config.py` 包含 MCP 服务专属配置：协议版本、超时、工具白名单、安全策略；
- 所有依赖**仅注入给 MCP 服务**，保证服务独立性。

### 5. MCP 服务部署与运行

- 支持**纯 MCP 模式启动**：直接作为标准 MCP 服务供 AI 调用
- 支持**MCP+HTTP 双模式**：同时提供 OpenAPI 接口与 MCP 服务
- 容器化打包时，**优先暴露 MCP 服务端口与协议**

## 四、R（Result）结果：MCP 服务架构落地价值

1. **标准 MCP 服务可用**
    
    实现了完全兼容 MCP 协议的独立服务，任何支持 MCP 的 AI 客户端可直接连接调用。
    
2. **MCP 服务高度可扩展**
    
    工具化设计让 MCP 服务可无限扩展能力，无需重构协议层。
    
3. **MCP 服务安全可控**
    
    在协议层实现权限控制、指令限制，满足企业级安全要求。
    
4. **MCP 服务可独立交付**
    
    可作为通用组件嵌入任何 AI 系统，实现 “一次开发、随处集成”。
    
5. **架构价值聚焦 MCP**
    
    所有技术栈均服务于 MCP 能力，轻量化、高性能、无冗余，完美适配 AI 生态。

### 总结

本项目**以 MCP 服务为绝对核心**，采用标准协议层、工具扩展层、数据支撑层的架构，实现了**轻量化、标准化、可插拔、安全可控**的 MCP 服务，成为 AI 模型与外部系统之间的官方、统一、可靠的交互入口。

### MCP 服务项目阐述（STAR 法则）

#### 一、Situation（背景）

在基于 FastAPI 构建的教育信息查询系统中，核心诉求是实现**强基计划高校信息的智能化查询**，具体需支持两种核心场景：一是通过 SQL 直接查询吉林大学强基计划相关数据，二是将用户自然语言问题转化为 SQL 并执行，最终返回精准结果。

项目面临的关键挑战包括：

1. 需适配异步通信模式，保证 MCP（Model Context Protocol）服务的流数据传输稳定性；
2. 需打通 “自然语言→SQL→数据查询” 的全链路，确保非技术用户也能通过日常语言获取精准的高校强基计划信息；
3. 需通过自动化测试验证 MCP 服务的可用性，避免接口调用、工具调用（execute_sql/nl_to_sql_volc）环节出现异常。

#### 二、Task（任务）

作为核心开发人员，我的核心任务包括：

1. 基于 MCP 协议设计并实现强基计划信息查询的异步接口（`/further_qj/mcp`）；
2. 封装 MCP Client 的调用逻辑，支持`execute_sql`（SQL 直查）和`nl_to_sql_volc`（自然语言转 SQL）两个核心工具；
3. 编写自动化测试用例（pytest+anyio），验证 MCP 服务的接口可用性、工具调用正确性及数据返回准确性；
4. 保障异步流数据传输（streamablehttp_client）的稳定性，确保 ClientSession 初始化、工具调用、结果返回全流程无异常。

#### 三、Action（行动）

1. **技术选型与基础封装**：
    
    - 采用 FastAPI 作为 Web 框架，结合 anyio 实现异步接口开发，适配 MCP 协议的异步通信特性；
    - 封装`streamablehttp_client`工具，实现 MCP 服务端与客户端的流数据双向传输，确保会话（ClientSession）的初始化和销毁可控。
    
2. **核心功能开发**：
    
    - 实现`/further_qj/mcp`接口，承接客户端的 MCP 协议请求，解析工具调用参数（如 SQL 语句、自然语言查询文本）；
    - 对接`execute_sql`工具：接收 SQL 语句（如`SELECT * FROM qj_schools WHERE school_name LIKE '%吉林大学%'`），执行并返回吉林大学强基计划的学校 / 专业数据；
    - 对接`nl_to_sql_volc`工具：接收自然语言查询（如 “告诉我吉林大学强基计划有哪些专业？”），调用大模型将其转化为可执行 SQL，再执行并返回结果。
    
3. **自动化测试落地**：
    
    - 编写`test_mcp_sql_query`用例：验证 SQL 直查场景下，MCP 服务能正确接收 SQL 参数、执行查询并返回结构化结果；
    - 编写`test_mcp_nl_to_sql`用例：验证自然语言转 SQL 场景下，工具能正确解析用户意图、生成合法 SQL 并返回精准数据；
    - 引入日志（logging）和打印输出，便于测试过程中排查数据返回异常，确保接口状态码、返回数据结构符合预期。
    
4. **异常处理与稳定性保障**：
    
    - 确保 ClientSession 的异步上下文管理（`async with`），避免会话泄露；
    - 对工具调用的入参做合法性校验，对 SQL 执行、自然语言转 SQL 的异常场景做兜底处理。
    

#### 四、Result（结果）

1. **功能层面**：
    
    - 成功实现 MCP 服务的核心接口，支持 SQL 直查和自然语言查询两种模式，能精准返回吉林大学强基计划的学校、专业等信息，满足非技术用户的查询需求；
    - 异步流数据传输全程稳定，ClientSession 初始化 / 销毁成功率 100%，工具调用无超时、数据丢失问题。
    
2. **测试层面**：
    
    - 完成核心场景的自动化测试用例覆盖，测试通过率 100%，能快速验证接口变更后的功能正确性；
    - 通过日志和打印输出，实现测试过程的可观测性，降低问题排查成本。
    
3. **业务层面**：
    
    - 打通 “自然语言→SQL→数据” 的全链路，提升了强基计划信息查询的易用性，非技术人员无需了解 SQL 即可获取精准数据；
    - 接口具备可扩展性，可快速适配其他高校（如清华大学、北京大学）的强基计划信息查询需求。
