# 恒星分类机器学习项目（STAR 法则详述）

## S（Situation 背景 / 情境）

恒星、星系、类星体的自动分类是天文数据处理的核心任务，传统人工分类效率低、难以应对大规模观测数据。

我参与的项目面向**10 万条深空观测光谱数据**，包含 17 个原始特征，数据存在缺失值、异常值、高维度、特征强相关等问题，需要构建自动化分类模型提升效率与准确率。

## T（Task 任务 / 目标）

我的核心任务：

1. 完成**数据清洗、特征工程、模型搭建与调优**全流程
2. 构建高精度机器学习模型，实现恒星 / 星系 / 类星体三分类
3. 输出可落地、可评估的分类系统，并完成项目报告
    
    团队目标：测试集准确率≥94%，模型具备泛化能力与工程可用性。

## A（Action 行动 / 执行）

我按标准机器学习流程落地执行，关键动作如下：

1. **数据清洗**
    
    - 用 fillna 填充缺失值为 0
    - 用 3σ 原则剔除 / 修正异常值
    - 去重处理，保证样本唯一性
    
2. **数据集划分**
    
    - 按 7:1.5:1.5 随机切分：训练集、验证集、测试集
    
3. **特征工程（核心工作）**
    
    - 特征构造：r-g、r/g、r×g、统计特征、交互特征
    - 标准化与归一化，统一特征尺度
    - Pearson 相关性分析，处理多重共线性
    - 随机森林评估特征重要性，确定**redshift 为最强特征**
    - XGBoost 特征选择 + PCA 降维到 9 维
    
4. **模型搭建与调优**
    
    - 选用**随机森林**作为分类器，抗过拟合、适合高维数据
    - 用 Pipeline 整合：标准化→PCA→分类器
    - GridSearchCV 5 折交叉验证，搜索最优超参
    - 最优配置：PCA=9，n_estimators=250，max_depth=None
    
5. **模型评估**
    
    - 用准确率、精确率、召回率、F1 全面评估
    - 分析类别表现，总结模型优缺点
    

## R（Result 结果 / 成果）

项目最终取得明确可量化成果：

1. **核心指标**
    
    - 验证集：Acc 0.944，Precision 0.9452，Recall 0.944，F1 0.9436
    - 测试集：Acc 0.9426，Precision 0.9436，Recall 0.9426，F1 0.9422
    
2. **工程价值**
    
    - 实现天文光谱数据自动化分类，效率远高于传统方法
    - 特征工程有效降低维度、提升模型稳定性
    - 完整可复现流程，可扩展到更大数据集
    
3. **个人产出**
    - 独立负责数据清洗、模型实现、报告撰写核心模块
    - 掌握数据清洗、特征工程、集成学习、超参搜索全流程
