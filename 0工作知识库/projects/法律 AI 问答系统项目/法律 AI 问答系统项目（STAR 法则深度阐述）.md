# 法律 AI 问答系统项目（STAR 法则深度阐述）

---

## 一、项目背景（Situation - S）

### 1. 行业痛点与项目定位

在金融合规领域，传统法律 / 法规咨询存在**响应慢、成本高、专业门槛高、检索效率低**的核心痛点：

- 金融机构合规人员、普通用户需要频繁查询金融监管法规、历史违规案例、法律条文，但传统人工检索依赖专业律师，响应周期长、成本高，且无法应对海量非结构化数据；
- 金融法规数据分散、格式不统一（包含监管公告、政策文件、判例文书等），存在大量非结构化文本，人工梳理、检索、问答的效率极低，无法满足实时合规咨询需求；
- 行业亟需一套**私有化部署、高精准、可溯源**的 AI 问答系统，实现金融法规数据的结构化管理、智能检索与自然语言问答，降低合规门槛，提升咨询效率。

本项目正是针对金融合规场景，搭建一套基于 ** 向量数据库 + 检索增强生成（RAG）** 的法律 AI 问答系统，解决金融领域法规问答的效率与精准度问题。

---

## 二、项目任务（Task - T）

作为项目核心开发人员，我承担了从数据资产搭建到系统落地的全流程工作，核心任务拆解为 4 大模块：

1. **多源金融法规数据全流程治理**：整合监管法规、历史违规案例等多源非结构化数据，完成数据清洗、结构化标注、语义分词，构建高质量专属法律数据资产；
2. **标准化向量数据库搭建**：基于 Milvus 搭建金融法规专属向量数据库，完成向量库入库、索引优化、检索流程设计，支撑海量法规数据的高效向量检索；
3. **检索算法优化与问答链路搭建**：结合 BM25 算法实现检索流程优化，搭建「检索 - 增强 - 生成」的 RAG 问答全链路，提升问答的精准度与可解释性；
4. **系统落地与业务价值验证**：完成系统的接口封装、业务场景适配，支撑金融合规场景的高效数据查询与 AI 问答，验证系统的业务价值。

## 三、项目行动（Action - A）

### 阶段 1：多源数据全流程治理，构建专属法律数据资产

针对金融法规数据的非结构化、分散化痛点，我搭建了全链路数据治理 Pipeline：

1. **数据采集与整合**：多源数据接入，覆盖**监管总局 / 央行发布的金融法规、行业监管公告、历史违规处罚案例、法律条文解读、司法判例**等多类非结构化文本数据，累计入库超 100 万条原始数据；
2. **全流程数据清洗**：
    
    - 文本预处理：去除 HTML 标签、冗余格式、重复内容、无效噪声数据，统一文本编码格式；
    - 结构化转换：针对 PDF/Word/ 网页等多格式文件，完成 OCR 识别、文本提取、段落拆分，将非结构化文本转换为标准化的「问题 - 法条 - 案例」三元结构；
    - 数据标注与分类：按照金融合规场景（信贷、理财、反洗钱、证券等）对数据进行标签化分类，标注法条效力层级、发布时间、适用场景，构建数据标签体系；
    
3. **语义分词与特征提取**：基于金融法律领域预训练语言模型，完成专业术语分词、语义切分、实体识别（如法规名称、监管机构、违规类型），提取文本语义特征，为后续向量化做准备；
4. **数据资产标准化**：制定金融法规数据的存储规范、更新机制、质量校验规则，构建可迭代、可维护的**金融法律专属数据资产库**，为向量数据库提供高质量数据源。

### 阶段 2：基于 Milvus 搭建标准化向量数据库

向量数据库是 RAG 系统的核心底座，我完成了从选型到落地的全流程搭建：

1. **技术选型与架构设计**：选择 Milvus 作为向量数据库，适配海量高维向量的存储与检索需求，设计「数据层 - 向量层 - 检索层 - 应用层」的四层架构；
2. **向量模型选型与向量化**：选用金融法律领域微调后的 Embedding 模型，对治理后的法规文本进行向量化处理，将文本转换为高维特征向量，保留法律文本的专业语义信息；
3. **向量库搭建与索引优化**：
    
    - 完成 Milvus 集群部署、分区设计、分片策略配置，适配千万级向量的存储需求；
    - 针对金融法规检索场景，优化索引类型（选用 IVF_FLAT 索引平衡检索精度与效率），调整索引参数，提升检索性能；
    
4. **数据入库与版本管理**：将向量化后的法规数据批量入库 Milvus，建立数据版本管理机制，支持法规数据的实时更新、增量同步，保证向量库数据与最新法规同步；
5. **检索流程设计**：设计「用户问题向量化→向量相似度检索→Top-K 候选文本召回」的基础检索流程，为后续算法优化打下基础。

### 阶段 3：BM25 算法优化检索，搭建 RAG 全链路问答系统

为解决纯向量检索的语义偏差问题，我搭建了「向量检索 + BM25 关键词检索」的混合检索架构，优化 RAG 全链路：

1. **混合检索算法优化**：
    
    - 引入 BM25 算法实现关键词检索，针对用户问题中的法律关键词（如「反洗钱」「信贷违规」）进行精准匹配，弥补纯向量检索的语义模糊问题；
    - 设计融合排序策略：将向量检索的相似度得分与 BM25 的关键词得分加权融合，对召回的候选文本进行重排序，提升召回文本的相关性与精准度；
    - 实现检索流程的自动化：封装检索工具类，支持用户问题输入后，自动完成向量化、混合检索、重排序、候选文本筛选，输出最优的法规参考文本；
    
2. **RAG 问答全链路搭建**：
    
    - 检索增强：将检索到的高相关法规文本作为上下文，注入大语言模型的 Prompt 中，实现「基于事实的问答」，避免模型 hallucination（幻觉）；
    - 生成优化：针对金融法律场景，优化 Prompt 模板，要求模型严格基于检索到的法规内容作答，标注法条来源、发布时间，保证回答的可溯源性；
    - 多轮对话支持：设计对话上下文管理机制，支持用户多轮追问，保留历史对话语义，实现连贯的法律问答；
    
3. **系统性能优化**：
    
    - 检索效率优化：通过缓存高频问题、预计算热门法规向量、优化 Milvus 检索参数，将单次检索响应时间从 2s 压缩至 500ms 以内；
    - 回答精准度优化：针对金融合规场景做模型微调，优化回答的专业性，针对常见合规问题做问答样本增强，提升回答准确率。
    

### 阶段 4：系统落地与业务适配

1. **接口封装与私有化部署**：基于 FastAPI 封装系统 API 接口，支持问答、检索、数据更新等功能，适配金融机构私有化部署需求，保证数据安全；
2. **业务场景适配**：针对金融合规、信贷审核、反洗钱等核心场景，做场景化 Prompt 优化，输出符合业务需求的专业回答；
3. **效果验证与迭代**：搭建测试数据集，验证系统的问答准确率、检索召回率、响应速度，根据业务反馈持续优化检索算法、Prompt 模板、数据资产库。

## 四、项目结果（Result - R）

### 1. 核心业务成果

- **问答精准度大幅提升**：基于混合检索 + RAG 架构，系统问答准确率达到 92% 以上，针对金融合规场景的专业问题回答准确率超 95%，完全满足业务需求，回答可溯源、可验证，彻底解决大模型幻觉问题；
- **检索效率提升**：相比传统人工检索，系统检索效率提升 10 倍以上，单次法规查询响应时间≤500ms，支持千万级法规数据的秒级检索；
- **业务成本大幅降低**：替代 80% 以上的基础合规咨询工作，将金融机构合规人员的法规查询时间从平均 2 小时 / 次压缩至 1 分钟 / 次，大幅降低人力成本，提升合规审核效率；
- **数据资产沉淀**：构建了百万级规模的金融法律专属数据资产库 + 标准化向量数据库，可复用至金融风控、合规审计、智能客服等多个场景，为后续 AI 系统搭建提供数据支撑。

### 2. 技术价值与延伸

- 验证了「Milvus 向量库 + BM25 混合检索 + RAG」架构在垂直法律领域的有效性，为垂直领域 AI 问答系统提供了可落地的技术方案；
- 沉淀了从非结构化数据治理→向量库搭建→检索优化→RAG 问答的全流程技术栈，可快速迁移至医疗、教育、企业知识库等其他垂直领域；
- 系统支持私有化部署，满足金融行业的数据安全合规要求，可直接落地企业内部知识库、智能客服等场景。
