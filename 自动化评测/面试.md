### 🎯 一、岗位核心能力拆解

这个岗位是**大模型自动化评测与数据有效性验证**方向，核心是「用代码和 AI 技术，给大模型做 “体检” 和 “数据筛选”」，你需要围绕这 4 个核心模块准备：

---

### 🧩 二、技术栈准备清单

#### 1. 编程语言与工程能力

- **Python**：必须精通，重点准备：
    
    - 常用库：`pandas`/`numpy`（数据处理）、`requests`/`FastAPI`（接口开发）、`pytest`（自动化测试）、`transformers`/`accelerate`（大模型调用）
    - 代码规范：PEP8、单元测试、Git 协作、Docker 容器化部署
    
- **Linux 与 DevOps**：
    
    - 常用命令：文件操作、进程管理、环境部署
    - Docker：镜像构建、容器运行、`Dockerfile`编写
    
- **算法基础**：
    
    - 常见机器学习算法（分类 / 回归 / 聚类）、深度学习基础（Transformer 架构）
    - 提示工程（Prompt Engineering）、模型调优（LoRA/QLoRA 微调）
    

#### 2. 大模型评测与数据能力

- **评测体系**：
    
    - 熟悉主流评测集：MMLU、GSM8K、C-Eval、HumanEval、MT-Bench 等
    - 掌握评测方法：自动评测（BLEU/Rouge/Perplexity）、人工评测、对抗评测
    - 能讲出：如何设计一个垂类领域（如医疗 / 金融）的评测集
    
- **数据有效性验证**：
    
    - 如何通过模型训练效果反推数据质量
    - 用 Agent / 工具自动化判别脏数据、低质量数据
    - 数据清洗、去重、筛选的工程实践
    

#### 3. 工具与平台开发

- 能讲出：你如何搭建一个自动化评测流水线（数据加载 → 模型推理 → 结果打分 → 报告生成）
- 了解：大模型推理框架（vLLM、Text Generation Inference）、评测平台架构
- 加分项：参与过开源评测工具 / 平台的开发或二次开发

#### 4. 科研与探索能力

- 能讲出：你读过的大模型评测 / 数据治理相关顶会论文（如 ACL/NeurIPS/ICML）
- 准备 1 个你做过的「自建评测集 / 数据验证」项目，说明思路、方法和成果
- 了解：如何将工程实践转化为学术论文（实验设计、结果分析、写作框架）

### 💼 三、项目与经历准备

你需要准备**2-3 个能匹配岗位的项目**，用 STAR 法则（Situation-Task-Action-Result）讲述：

1. **自动化评测项目**：
    
    - 背景：为某大模型做垂类能力评测
    - 任务：搭建自动化评测平台，集成 X 个评测集
    - 行动：用 Python+FastAPI 开发接口，对接模型推理服务，实现自动打分
    - 结果：评测效率提升 X 倍，发现模型在 X 场景下的短板，助力模型迭代
    
2. **数据有效性验证项目**：
    
    - 背景：训练数据质量差导致模型效果波动
    - 任务：设计数据筛选方案，剔除无效数据
    - 行动：构建 Agent 自动校验数据逻辑一致性，用模型微调前后效果对比验证
    - 结果：训练数据量减少 X%，模型指标提升 X%

### 🗣️ 四、面试高频问题准备

#### 1. 技术问题

- 「你了解哪些大模型自动评测方法？它们的优缺点是什么？」
- 「如何判断一段训练数据是有效的？有哪些自动化手段？」
- 「如果让你从零搭建一个大模型评测平台，你会怎么设计？」
- 「Python 中如何优化大模型推理的并发性能？」
- 「Docker 在你的项目中起到了什么作用？」

#### 2. 项目问题

- 「你在这个项目中遇到的最大技术难点是什么？怎么解决的？」

- 「如果再做一次这个项目，你会做哪些改进？」

- 「你如何和算法 / 产品团队协作，推动评测结果落地？」
