- 插入 / Upsert / 删除、批量导入、分区策略、Compaction、TTL、数据一致性与 WAL
## 一、面试极简知识点

- **插入**：新增数据，主键重复会报错。
- **Upsert**：有则更新、无则插入，最常用。
- **删除**：按主键 / 过滤条件删除，标记删除不立即清理。
- **批量导入**：大批量数据高速导入，比单条插入快。
- **分区策略**：按时间 / 业务分区，查询只扫指定分区，提速。
- **Compaction**：合并小数据段、清理删除数据，提升查询性能。
- **TTL**：自动过期删除，节省存储。
- **数据一致性**：读取最新 / 已落盘数据的级别选择。
- **WAL**：写前日志，防止宕机丢数据。

## 二、综合示例

日志向量库：

1. 批量导入历史日志；
2. 按天分 Partition；
3. 实时 Upsert 增量日志，WAL 保证不丢；
4. 设置 TTL 自动删除 30 天前数据；
5. 后台 Compaction 合并碎片；
6. 查询指定日期分区，保证一致性读取。

## 三、一句话总结

插入 / Upsert / 批量导入负责写入，分区 / TTL/Compaction 优化存储，WAL 与一致性保障数据安全可靠。
