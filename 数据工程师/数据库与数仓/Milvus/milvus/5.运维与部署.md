- 单机 / 集群部署、K8s 与容器化、监控告警、性能调优（内存 / CPU/IO）、高可用与故障恢复
## 一、面试极简知识点

- **单机部署**：测试 / 轻量场景，一键启动，资源需求低。
- **集群部署**：生产环境，多节点分布式，支持水平扩容。
- **K8s 与容器化**：使用 Docker + Helm 部署，便于编排、升级、扩缩容。
- **监控告警**：对接 Prometheus + Grafana，监控 QPS、延迟、磁盘、连接数。
- **性能调优**：内存控制 Segment 缓存；CPU 优化索引并发；IO 提升批量写入与查询吞吐。
- **高可用**：多副本、节点冗余，避免单点故障。
- **故障恢复**： WAL 恢复、元数据备份、节点重启自愈、数据重平衡。

## 二、综合示例

生产向量知识库：

1. K8s + Helm 部署 Milvus 集群，多节点多副本；
2. 配置 Prometheus 监控查询延迟、内存使用率；
3. 根据并发调大 CPU 线程、控制 HNSW 内存占用；
4. 开启 WAL 与自动备份；
5. 某节点宕机后自动故障转移，数据不丢失、服务不中断。

## 三、一句话总结

以容器 / K8s 实现标准化部署，通过监控与资源调优保障性能，用多副本、WAL、备份实现高可用与故障恢复。
