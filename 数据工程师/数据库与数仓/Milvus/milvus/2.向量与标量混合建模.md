- 向量维度、距离度量（L2/IP/Cosine）、主键 / 自动 ID、多向量字段、JSON / 数组等高级类型Milvus
# 一、面试简要知识点（精简背诵版）

- **向量维度**：Embedding 模型输出的固定长度（如 768/1024），建库必须指定。
- **距离度量**：L2（欧式距离，越小越相似）、IP（内积，越大越相似）、Cosine（余弦相似度，归一化后用）。
- **主键 / 自动 ID**：唯一标识数据，支持手动指定主键、Milvus 自动生成 ID。
- **多向量字段**：一个集合支持多个向量列（如文本向量 + 图像向量）。
- **高级类型**：JSON（存结构化属性）、数组（存多标签 / 多值），用于标量过滤。

# 二、综合示例（串联所有概念）

建一个**商品检索集合**：

- 向量 1：`text_emb`(768 维，Cosine) → 商品标题向量
- 向量 2：`img_emb`(512 维，L2) → 商品图片向量
- 主键：`prod_id`(手动指定)
- 标量：`price`(int)、`tags`(数组)、`ext_info`(JSON)
    
    一条数据就是**多向量 + 多类型标量**的混合实体。

# 三、一句话总结

向量指定维度与距离，主键唯一标识，多向量 + JSON / 数组实现向量检索与标量过滤的混合建模。
