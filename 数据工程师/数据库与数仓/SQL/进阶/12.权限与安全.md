# 12. 权限与安全（面试极简背诵版）

## 一、核心知识点

1. **GRANT / REVOKE**
    
    - GRANT：赋权（查询、修改、管理）
    - REVOKE：回收权限
    - 标准权限控制语法
    
2. **列权限**
    
    - 只开放**部分字段**，屏蔽敏感字段（手机号、身份证）
    
3. **行权限**
    
    - 只允许查看**符合条件的行**（如只能看自己部门数据）
    
4. **数据脱敏**
    
    - 敏感数据**掩码显示**（138****1234）
    - 加密存储，防止泄露
    - 中台 / 治理**必考**
    

---

## 二、综合示例

### 1. 赋权与回收

```sql
-- 赋权查询
GRANT SELECT ON dwd_user TO user_analyst;

-- 赋权指定列
GRANT SELECT(user_id, name) ON dwd_user TO user_analyst;

-- 回收权限
REVOKE SELECT ON dwd_user FROM user_analyst;
```

### 2. 行权限示例

```sql
-- 只允许查看北京地区数据
CREATE ROW POLICY filter_beijing
ON dwd_user
FOR SELECT
WHERE city = '北京';
```

### 3. 脱敏函数（掩码）

```sql
-- 手机号脱敏
SELECT 
  user_id,
  -- 掩码：138****1234
  concat(substr(phone,1,3),'****',substr(phone,8,4)) AS phone_mask,
  -- 身份证脱敏
  concat(substr(id_card,1,6),'********',substr(id_card,15,4)) AS id_mask
FROM dwd_user;
```

---

## 三、一句话总结

GRANT/REVOKE 做基础权限控制，列权限屏蔽敏感字段，行权限限制数据范围，脱敏函数实现敏感数据掩码保护。

### 总结

1. **列权限**：屏蔽敏感字段
2. **行权限**：控制可见行范围
3. **数据脱敏**：治理合规必备
4. **生产必做**：最小权限原则 + 敏感数据脱敏
