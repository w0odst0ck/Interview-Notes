# 数据质量规则（面试极简版，直接背）

## 一、核心知识点

1. **空值校验**
    
    检查关键字段（主键、外键、指标字段）是否为 NULL 或空字符串，保证数据非空完整性。
    
2. **唯一性校验**
    
    检查主键、唯一键、业务单号是否重复，保证数据唯一不重复。
    
3. **范围校验**
    
    检查数值、日期、枚举值是否在合理区间，避免异常值、脏数据。
    
4. **重复数据排查**
    
    找出完全重复或业务逻辑重复的记录，用于去重清洗。
    

---

## 二、综合示例（一套 SQL 全覆盖）

以表 `dwd_order_detail` 为例：

order_id（订单 ID）、user_id（用户 ID）、pay_amount（支付金额）、pay_time（支付时间）、dt（日期分区）

### 1. 空值校验

```sql
SELECT
  COUNT(1) AS total,
  SUM(CASE WHEN order_id IS NULL THEN 1 ELSE 0 END) AS order_id_null_cnt,
  SUM(CASE WHEN pay_amount IS NULL THEN 1 ELSE 0 END) AS amount_null_cnt
FROM dwd_order_detail
WHERE dt = '2025-04-01';
```

### 2. 唯一性校验

```sql
SELECT order_id, COUNT(1) AS cnt
FROM dwd_order_detail
WHERE dt = '2025-04-01'
GROUP BY order_id
HAVING COUNT(1) > 1;
```

### 3. 范围校验

```sql
SELECT *
FROM dwd_order_detail
WHERE dt = '2025-04-01'
  AND pay_amount < 0          -- 金额非法
  AND pay_time > '2025-04-01' -- 时间越界
  AND order_status NOT IN ('0','1','2','3'); -- 枚举非法
```

### 4. 重复数据排查

```sql
SELECT order_id, user_id, pay_time, COUNT(1) AS dup_cnt
FROM dwd_order_detail
WHERE dt = '2025-04-01'
GROUP BY order_id, user_id, pay_time
HAVING COUNT(1) > 1;
```

---

## 三、一句话总结

空值校验保完整性，唯一性校验防主键重复，范围校验过滤脏数据，重复排查保证数据明细唯一，是数据治理与中台建设的基础质量手段。
