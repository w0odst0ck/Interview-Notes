# 数据治理 SQL 核心（面试必背・极简版）

## 一、核心知识点

1. **数据血缘**
    
    追踪数据**从哪来、到哪去、经过哪些加工**，表级别 / 字段级别血缘。
    
2. **字段血缘查询**
    
    定位字段的**源头表、源头字段、加工逻辑、下游表**，治理、回溯必备。
    
3. **元数据统计**
    
    统计表 / 字段基础信息：表大小、行数、分区、字段类型、注释、创建时间。
    
4. **数据质量校验 SQL**
    
    用 SQL 做自动化质量检查，治理核心动作。
    

---

## 二、质量校验 4 大必查项（面试 + 工作高频）

1. **空值校验**：关键字段不能为空
2. **唯一性校验**：主键 / 唯一键不能重复
3. **合法性校验**：枚举值、范围、格式合规
4. **完整性校验**：日分区数据量无断崖缺失

---

## 三、综合示例（直接背）

### 1. 空值校验

```sql
SELECT COUNT(*) null_cnt
FROM dwd_user
WHERE user_id IS NULL;
```

### 2. 唯一性校验（主键重复检查）

```sql
SELECT user_id, COUNT(*) cnt
FROM dwd_user
GROUP BY user_id
HAVING COUNT(*) > 1;
```

### 3. 枚举值合法性校验

```sql
SELECT status, COUNT(*) cnt
FROM dwd_order
WHERE status NOT IN ('1','2','3');
```

### 4. 分区数据完整性（日环比）

```sql
SELECT dt, COUNT(*) cnt
FROM dwd_order
GROUP BY dt
ORDER BY dt DESC LIMIT 7;
```

### 5. 元数据统计（Hive）

```sql
SHOW TBLPROPERTIES dwd_user;
DESCRIBE FORMATTED dwd_user;
```

---

## 三、一句话总结

数据血缘追踪流向，字段血缘定位来源，元数据统计资产信息，数据质量 SQL 做空值、重复、合法性、完整性四大校验。

### 总结

1. **治理三件套**：血缘 + 元数据 + 数据质量
2. **质量必查 4 项**：空值、重复、合法性、完整性
3. **面试一句话**：数据治理就是通过血缘可追溯、元数据可管理、质量可监控，保障数据准确、可用、合规。
