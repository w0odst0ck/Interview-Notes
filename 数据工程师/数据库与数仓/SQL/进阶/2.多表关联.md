# 多表关联（JOIN）面试精简版

## 一、核心知识点（必背）

1. **INNER JOIN（内连接）**
    
    - 只返回**两张表都匹配**的数据
    - 取**交集**
    
2. **LEFT JOIN（左连接）**
    
    - 以**左表为基准**，返回左表所有数据 + 右表匹配数据
    - 右表无匹配则显示 NULL
    - 数仓 / 中台**最常用**
    
3. **RIGHT JOIN（右连接）**
    
    - 以**右表为基准**，返回右表所有数据 + 左表匹配数据
    - 实际工作中很少用，换成 LEFT JOIN 即可
    
4. **FULL JOIN（全连接）**
    
    - 返回**两张表所有数据**，匹配不上的补 NULL
    - 取**并集**
    
5. **关联条件（ON）**
    
    - 写在 `ON` 后，不是 WHERE
    - 必须明确**关联键**（如 user_id、order_id）
    
6. **关联键设计**
    
    - 字段类型必须一致（int 不能连 string）
    - 优先用**主键 / 外键**（唯一、稳定）
    - 避免 NULL 值关联（会丢失数据）
    - 少用多字段关联，尽量单字段关联

## 二、综合示例（串联所有知识点）

### 表 1：用户表 user_info

|user_id|user_name|
|---|---|
|1001|张三|
|1002|李四|
|1003|王五|

### 表 2：订单表 order_info

|order_id|user_id|amount|
|---|---|---|
|1|1001|99|
|2|1001|199|
|3|1004|299|

### 1. INNER JOIN（交集）

```sql
SELECT u.user_id, u.user_name, o.order_id, o.amount
FROM user_info u
INNER JOIN order_info o 
ON u.user_id = o.user_id;
```

结果：只返回**两边都有的 1001**

### 2. LEFT JOIN（左表全量）

```sql
SELECT u.user_id, u.user_name, o.order_id, o.amount
FROM user_info u
LEFT JOIN order_info o 
ON u.user_id = o.user_id;
```

结果：返回**所有用户**，没订单的显示 NULL

### 3. RIGHT JOIN（右表全量）

```sql
SELECT u.user_id, u.user_name, o.order_id, o.amount
FROM user_info u
RIGHT JOIN order_info o 
ON u.user_id = o.user_id;
```

结果：返回**所有订单**，1004 无用户显示 NULL

### 4. FULL JOIN（全量）

```sql
SELECT u.user_id, u.user_name, o.order_id, o.amount
FROM user_info u
FULL JOIN order_info o 
ON u.user_id = o.user_id;
```

结果：**所有用户 + 所有订单**

---

## 三、一句话总结

INNER 取交集、LEFT 以左表为准全量返回、RIGHT 以右表为准、FULL 取并集，关联必须用同类型稳定键，**数仓优先用 LEFT JOIN**。

### 总结

1. 四种 JOIN 核心就是**交集、左全量、右全量、并集**
2. 工作中**90% 用 LEFT JOIN**
3. 关联键要**类型一致、非空、稳定**
