# 数据建模 SQL（数据中台 / 治理 核心必考）

## 一、核心知识点（必背）

### 1. 数仓分层建模（ODS/DWD/DWS/ADS）

- **ODS（原始数据层）**：直接同步源数据，不清洗，保留原样
- **DWD（明细数据层）**：清洗、去重、关联、标准化，**明细宽表**
- **DWS（汇总数据层）**：按主题轻度聚合，**日汇总、用户汇总、主题宽表**
- **ADS（应用数据层）**：面向报表 / 接口，**最终指标表**

### 2. 拉链表（数仓必用）

- 记录**历史全量状态**，按时间区间存储变化数据
- 关键字段：`start_date`/`end_date`，`9999-12-31`表示当前有效
- 场景：用户信息、订单状态、账户信息变化追踪

### 3. 快照表

- 每天全量存储**当天状态**，按日期分区
- 场景：每日库存、每日资产、每日用户画像

### 4. 缓慢渐变维（SCD）

- 处理维度表**缓慢变化**（如用户修改手机号、地址）
- 常用：**SCD2**（保留历史版本，用生效 / 失效时间标识）

## 二、综合示例（串联所有）

### 1. 分层建表示例

```sql
-- ODS 原始层（直接同步）
CREATE TABLE ods_user (id INT,name STRING,phone STRING,dt STRING);

-- DWD 明细层（清洗+标准化）
CREATE TABLE dwd_user_info (id INT,name STRING,phone STRING,create_time STRING)
PARTITIONED BY (dt STRING) STORED AS ORC;

-- DWS 汇总层（用户日汇总）
CREATE TABLE dws_user_day (user_id INT,order_cnt INT,pay_amt DECIMAL)
PARTITIONED BY (dt STRING) STORED AS ORC;

-- ADS 应用层（报表指标）
CREATE TABLE ads_user_report (dt STRING,user_cnt INT,pay_amt DECIMAL);
```

### 2. 拉链表（SCD2 实现）

```sql
CREATE TABLE dim_user_zip (
  user_id INT,
  user_name STRING,
  start_date STRING,  -- 生效日期
  end_date STRING,    -- 失效日期
  is_current TINYINT  -- 1=当前有效
) STORED AS ORC;
```

### 3. 快照表

```sql
CREATE TABLE dim_user_snapshot (
  user_id INT,
  user_name STRING,
  phone STRING
) PARTITIONED BY (dt STRING) STORED AS ORC;  -- 每日全量快照
```

## 三、一句话总结

ODS/DWD/DWS/ADS 是数仓标准分层，拉链表与 SCD2 用于追踪历史状态，快照表存储每日全量状态，是数据中台建模核心。

### 总结

1. **四层分层**：中台数仓标准架构
2. **拉链表**：存储历史状态，治理必备
3. **SCD2**：缓慢渐变维标准方案
4. **快照表**：每日全量，简单高效
