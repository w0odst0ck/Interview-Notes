# 子查询与 CTE 面试精简版

## 一、核心知识点

1. **子查询**

- 嵌套在 `SELECT/INSERT/UPDATE/DELETE` 内部的查询语句
- 按结果分为：标量、单行、多行、表子查询

2. **标量子查询**

- 结果**只返回一行一列**（单个值）
- 可用在 `SELECT 字段、WHERE、HAVING` 中当作常量使用

3. **关联子查询（相关子查询）**

- 子查询**依赖外层表字段**才能执行
- 逐行匹配，常用于 “每行都要查一次附属信息” 场景

4. **CTE（WITH 子句）**

- 命名临时结果集，语法 `WITH 别名 AS (子查询)`
- 结构清晰、可复用、可读性远强于多层嵌套子查询
- 支持链式 CTE、递归 CTE（数仓常用）

## 二、综合示例

### 表结构

- user（用户）：user_id, user_name
- order（订单）：order_id, user_id, amount, create_time

### 1. 标量子查询

需求：查询所有订单，并附带**全平台订单总金额**

```sql
SELECT
  order_id,
  user_id,
  amount,
  (SELECT SUM(amount) FROM order) AS total_amount
FROM order;
```

### 2. 关联子查询

需求：查询每个用户**最新一笔订单**

```sql
SELECT
  u.user_id,
  u.user_name,
  o.amount
FROM user u
LEFT JOIN order o 
  ON u.user_id = o.user_id
WHERE o.create_time = (
  SELECT MAX(create_time) 
  FROM order 
  WHERE user_id = u.user_id  -- 关联外层 u.user_id
);
```

### 3. CTE（WITH）

需求：先统计用户总消费，再筛选消费 > 500 的用户

```sql
WITH user_total AS (
  SELECT 
    user_id,
    SUM(amount) AS total
  FROM order
  GROUP BY user_id
)
SELECT
  u.user_id,
  u.user_name,
  ut.total
FROM user u
JOIN user_total ut ON u.user_id = ut.user_id
WHERE ut.total > 500;
```

---

## 三、一句话总结

标量子查询返回单个值，关联子查询依赖外层表逐行匹配，CTE 用 WITH 定义临时结果集，让复杂查询结构更清晰、易维护。
