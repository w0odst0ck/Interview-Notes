# Hive SQL 核心（数据中台 / 治理工程师必考）

## 一、核心知识点（必背）

1. **分区（PARTITION）**

- 按**日期 / 区域 / 业务线**拆分表目录，**减少全表扫描**
- 分**静态分区**（手动指定）、**动态分区**（自动匹配）
- 数仓必用：`dt`（日期）、`hour`（小时）

2. **分桶（BUCKET）**

- 对字段**哈希取模**拆分文件，优化**JOIN / 抽样**
- 适合大表关联，提升查询效率

3. **列存格式（ORC/Parquet）**

- **列式存储**，压缩比高、查询快
- 生产环境**强制使用**，替代文本格式

4. **UDF（自定义函数）**

- 内置函数不够时，**自己写函数**（字符串 / 脱敏 / 规则计算）
- 分：UDF（一行进一行出）、UDAF（聚合）、UDTF（一行进多行出）

5. **侧视图（LATERAL VIEW）**

- 配合 **explode** 把**数组 / Map 展开成多行**
- 行转列必备，处理 JSON、日志字段

## 二、综合示例（串联所有）

### 1. 建表（分区 + 分桶 + ORC）

```sql
CREATE TABLE user_behavior (
    user_id BIGINT,
    item_id ARRAY<STRING>,  -- 浏览商品ID数组
    behavior STRING
)
-- 分区
PARTITIONED BY (dt STRING)
-- 分桶
CLUSTERED BY (user_id) INTO 10 BUCKETS
-- 列存格式
STORED AS ORC
LOCATION '/user/hive/warehouse/user_behavior';
```

### 2. LATERAL VIEW + explode（行转列）

```sql
SELECT
    user_id,
    single_item_id,
    behavior,
    dt
FROM user_behavior
-- 炸开数组
LATERAL VIEW explode(item_id) tmp_table AS single_item_id
WHERE dt = '2025-08-01';
```

### 3. 动态分区插入

```sql
SET hive.exec.dynamic.partition=true;
SET hive.exec.dynamic.partition.mode=nonstrict;

INSERT INTO TABLE user_behavior PARTITION(dt)
SELECT user_id, item_id, behavior, dt FROM ods_behavior;
```

## 三、一句话总结

分区用于按时间 / 维度裁剪数据提速，分桶优化大表关联，ORC/Parquet 是生产标准存储，UDF 实现自定义逻辑，LATERAL VIEW 用于数组 / Map 行转列。

### 总结

1. **分区**：数据中台最核心优化手段，按 dt 查询
2. **列存**：生产表必须 ORC/Parquet
3. **侧视图**：数据清洗、展开必备
4. **Hive = 数仓基础技能，必须熟练**
