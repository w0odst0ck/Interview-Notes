# 性能优化（面试高频・极简背诵版）

## 一、核心知识点

1. **执行计划**
    
    查看 SQL 执行步骤，定位慢查询原因（EXPLAIN）。
    
2. **索引**
    
    加速查询，数仓 Hive 中较少用，主要在 MySQL/PostgreSQL 使用。
    
3. **分区裁剪**
    
    查询时**只扫需要的分区**，不读全表，最有效优化手段。
    
4. **避免全表扫描**
    
    加 WHERE 条件、用分区、加索引，减少扫描数据量。
    
5. **数据倾斜**
    
    某 Key 数据量过大，导致个别 Task 卡死。
    
    解决方案：**加盐打散、参数调优、单独处理倾斜 Key**。
    

---

## 二、核心 SQL & 优化写法

### 1. 查看执行计划

```sql
EXPLAIN SELECT * FROM dwd_order WHERE dt='2025-04-01';
```

### 2. 分区裁剪（最常用）

```sql
-- 好：只查1天分区
SELECT * FROM dwd_order WHERE dt = '2025-04-01';

-- 坏：全表扫描
SELECT * FROM dwd_order;
```

### 3. 避免数据倾斜（加盐法）

```sql
-- 倾斜key加盐打散
SELECT 
  concat(user_id, '_', rand()) AS new_key, 
  count(*) 
FROM dwd_order 
GROUP BY concat(user_id, '_', rand());
```

### 4. 避免全表扫描

```
-- 必须加分区/过滤条件
WHERE dt >= '2025-04-01' AND status = '1'
```

---

## 三、一句话总结

性能优化核心：**先分区裁剪避免全表扫描，再用执行计划排查问题，最后通过加盐打散解决数据倾斜**。

### 总结

1. **分区裁剪**：数仓第一优化手段
2. **执行计划**：排查慢 SQL 必备
3. **数据倾斜**：中台面试必考
4. **全表扫描**：生产严禁出现
