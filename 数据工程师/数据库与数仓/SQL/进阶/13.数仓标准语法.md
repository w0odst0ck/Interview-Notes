# 13. 数仓标准语法（面试 + 工作通用极简版）

## 一、核心知识点

1. **建表语句**

- 标准 `CREATE TABLE`，指定字段、类型、注释
- 生产统一用 **外部表 EXTERNAL**，防止误删数据

2. **字段规范**

- 小写 + 下划线：`user_id, order_no, pay_amount`
- 禁止关键字、拼音、无意义缩写
- 统一类型：ID 用 bigint，金额用 decimal，时间用 string

3. **注释 COMMENT**

- 表必须加注释，每个字段必须加注释
- 数据治理、元数据管理必备

4. **约束**

- 数仓不常用主键 / 外键约束，靠 **程序保证 + 质量校验**
- 常用：非空、唯一（逻辑约束）

5. **分区表设计**

- 按 `dt`（日期）一级分区，`hour` 二级分区
- 必须分区，禁止无分区大表
- 存储格式统一 **ORC/Parquet**

---

## 二、标准建表示例（直接背）

```sql
CREATE EXTERNAL TABLE dwd_user_order_di (
    user_id         BIGINT COMMENT '用户ID',
    order_id        STRING COMMENT '订单编号',
    pay_amount      DECIMAL(16,2) COMMENT '支付金额',
    create_time     STRING COMMENT '创建时间 yyyy-MM-dd HH:mm:ss',
    is_paid         TINYINT COMMENT '是否支付 1-是 0-否'
) 
COMMENT '用户订单明细宽表'
PARTITIONED BY (dt STRING COMMENT '日期分区 yyyy-MM-dd')
STORED AS ORC
LOCATION '/warehouse/dwd/dwd_user_order_di';
```

---

## 三、一句话总结

数仓建表统一使用外部表，字段小写下划线 + 完整注释，按日期分区，存储格式用 ORC/Parquet，遵循规范保证可维护、可治理。
