- 增删改查（SELECT / INSERT / UPDATE / DELETE）、WHERE / GROUP BY / ORDER BY / HAVING
# 一、简要知识点（面试必背版）

## 1. SELECT（查询）

- 从表中**读取数据**，最核心语句。
- 可指定字段、去重（DISTINCT）、起别名（AS）。

## 2. FROM

- 指定**数据源表**。

## 3. WHERE

- **行过滤**，在分组前筛选数据。
- 支持：=、>、<、>=、<=、!=、IN、LIKE、BETWEEN、AND/OR/NOT。

## 4. GROUP BY

- **按字段分组**，用于聚合统计。
- 常搭配：COUNT、SUM、AVG、MAX、MIN。

## 5. HAVING

- **分组后过滤**，只能跟聚合函数一起用。
- 区别：WHERE 过滤原始行，HAVING 过滤分组结果。

## 6. ORDER BY

- 对结果**排序**，ASC 升序（默认）、DESC 降序。

## 7. INSERT（插入）

- 向表中**新增数据**。

## 8. UPDATE（更新）

- **修改已有数据**，必须加 WHERE，否则全表更新。

## 9. DELETE（删除）

- 删除行数据，同样建议加 WHERE。

# 二、综合示例（串联所有知识点）

假设有一张用户订单表 `order_info`：

```sql
order_id  INT,    -- 订单ID
user_id   INT,    -- 用户ID
amount    DECIMAL(10,2), -- 订单金额
pay_time  DATE,   -- 支付时间
status    VARCHAR(10) -- 订单状态：已支付/未支付
```

## 1. INSERT 插入测试数据

```sql
INSERT INTO order_info (order_id, user_id, amount, pay_time, status)
VALUES 
(1, 1001, 99.0, '2025-01-01', '已支付'),
(2, 1001, 199.0, '2025-01-02', '已支付'),
(3, 1002, 299.0, '2025-01-03', '未支付'),
(4, 1003, 399.0, '2025-01-04', '已支付');
```

## 2. UPDATE 修改未支付订单

```sql
UPDATE order_info
SET status = '已支付', amount = 259.0
WHERE order_id = 3;
```

## 3. SELECT + WHERE + GROUP BY + HAVING + ORDER BY 综合查询

**需求**：查询 2025 年 1 月已支付订单中，**总消费金额 ≥ 200 元**的用户，按总金额降序展示。

```sql
SELECT 
  user_id,
  COUNT(order_id) AS order_cnt,
  SUM(amount) AS total_amount
FROM order_info
WHERE 
  pay_time BETWEEN '2025-01-01' AND '2025-01-31'
  AND status = '已支付'
GROUP BY user_id
HAVING SUM(amount) >= 200
ORDER BY total_amount DESC;
```

# 三、面试高频考点一句话总结

1. **WHERE 和 HAVING 区别**：WHERE 分组前过滤，HAVING 分组后过滤。
2. **执行顺序**：FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY。
3. **UPDATE/DELETE 必须加 WHERE**，否则全表操作，生产事故。
