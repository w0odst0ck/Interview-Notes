- （LOAD、INSERT、Sqoop/DataX 对接）
## 一、简要面试介绍

- **LOAD**：直接移动 / 复制 HDFS 文件到 Hive 表目录，**速度最快、无计算**，适合已落地到 HDFS 的文本 / 格式文件。
- **INSERT**：通过 SQL 写入，支持**覆盖 / 追加**，常用于数仓层与层之间的 ETL 转换。
- **Sqoop**：Hive ↔ 关系型数据库（MySQL/Oracle）双向同步，传统离线同步主力。
- **DataX**：通用异构数据源同步工具，比 Sqoop 更稳定、支持更多数据源，生产主流。

---

## 二、综合示例（串联全流程）

```sql
-- 1. LOAD 导入：HDFS 文件直接加载到分区
LOAD DATA INPATH '/tmp/raw.log' 
OVERWRITE INTO TABLE ods.user_login PARTITION(dt='2025-04-13');

-- 2. INSERT 层间同步：ODS → DWD
INSERT OVERWRITE TABLE dwd.user_login_d PARTITION(dt='2025-04-13')
SELECT uid, ip, ts FROM ods.user_login WHERE dt='2025-04-13';

-- 3. Sqoop 从 MySQL 导入 Hive
sqoop import \
--connect jdbc:mysql://... \
--table user \
--hive-table ods.user \
--hive-partition-key dt \
--hive-partition-value 2025-04-13

-- 4. DataX 同步 MySQL → Hive（配置 json 执行）
python datax.py mysql2hive.json
```

---

## 三、一句话总结

Hive 数据导入导出以 **LOAD 高速入表、INSERT 层间清洗**为主，用 **Sqoop/DataX 对接 MySQL 等业务库** 完成全链路数据同步。
