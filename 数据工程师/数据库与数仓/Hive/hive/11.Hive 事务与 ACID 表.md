# 11. Hive 事务与 ACID 表（更新删除、增量数据处理）

## 一、简要面试介绍

- **ACID 表**：支持原子性、一致性、隔离性、持久性的 Hive 表，可实现 **UPDATE / DELETE / MERGE**。
- **适用场景**：订单状态更新、用户信息修改、**增量数据合并（SCD 缓慢变化维）**。
- **必须条件**：**外部表不可用，必须是内部表（托管表）**、ORC 存储、分桶、开启事务参数。
- **核心能力**：支持**增量数据 Upsert（有则更新、无则插入）**，是数仓实时 / 离线增量合并关键。

---

## 二、综合示例（串联知识点）

```sql
-- 开启事务配置
SET hive.support.concurrency = true;
SET hive.enforce.bucketing = true;
SET hive.txn.manager = org.apache.hadoop.hive.ql.lockmgr.DbTxnManager;

-- 建 ACID 表：内部表 + 分桶 + ORC
CREATE TABLE user_info (
    uid BIGINT, name STRING, status STRING
)
CLUSTERED BY (uid) INTO 10 BUCKETS
STORED AS ORC
TBLPROPERTIES ('transactional'='true');

-- 更新
UPDATE user_info SET status='active' WHERE uid=1001;

-- 删除
DELETE FROM user_info WHERE status='inactive';

-- 增量 Upsert（最常用）
MERGE INTO user_info a
USING user_incr b
ON a.uid = b.uid
WHEN MATCHED THEN UPDATE SET *
WHEN NOT MATCHED THEN INSERT VALUES *;
```

---

## 三、一句话总结

Hive ACID 表基于**内部表 + ORC + 分桶 + 事务**实现，支持 **UPDATE/DELETE/MERGE**，主要用于**增量数据 Upsert 与状态更新**。
