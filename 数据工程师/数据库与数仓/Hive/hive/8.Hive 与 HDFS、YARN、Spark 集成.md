## 一、简要面试介绍

- **Hive 与 HDFS**
    
    Hive 本身**不存储数据**，只管理元数据；实际表数据、分区数据全部以文件形式落在 **HDFS** 对应目录。HDFS 是 Hive 的默认存储层。
    
- **Hive 与 YARN**
    
    Hive 的计算任务（MapReduce/Spark）都会提交到 **YARN** 做资源调度，由 YARN 分配 Container 执行，控制并发与资源隔离。
    
- **Hive 与 Spark**
    
    Hive 默认执行引擎是 MapReduce，生产常用 **Spark 作为执行引擎** 替代 MR，速度更快；同时 Spark 也可直接读写 Hive 表，共用 Metastore 元数据。
    

---

## 二、综合示例（串联整个流程）

1. 建表时指定路径，数据实际存在 **HDFS**：
    
    ```sql
    CREATE EXTERNAL TABLE user_login (...) PARTITIONED BY (dt) LOCATION '/data/ods/user_login';
    ```
    
2. 执行查询时，Hive 将 SQL 编译为任务，提交给 **YARN** 申请资源。
3. 设置执行引擎为 Spark：

    ```sql
    SET hive.execution.engine = spark;
    SELECT uid, COUNT(*) FROM user_login WHERE dt='2025-04-01' GROUP BY uid;
    ```
    
4. **Spark** 从 YARN 拿到资源，从 HDFS 读取数据计算，结果写回 HDFS 对应表目录。
5. 整个过程：**Hive 负责 SQL 编译 → YARN 管资源 → Spark 做计算 → HDFS 存数据**。

---

## 三、一句话总结

Hive 基于 **HDFS 存储数据**，通过 **YARN 调度计算资源**，并可对接 **Spark 作为高性能执行引擎**，构成完整的数仓计算体系。
