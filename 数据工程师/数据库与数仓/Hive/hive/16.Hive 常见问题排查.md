# 16. Hive 常见问题排查（执行慢、内存溢出、数据丢失、元数据不一致）

## 一、简要面试介绍

- **执行慢**：多由全表扫描、数据倾斜、Shuffle 过大、小文件过多、引擎效率低导致。
- **内存溢出（OOM）**：Map/Reduce 内存不足，SQL 逻辑复杂，小文件过多导致并发过高，或 Join 产生笛卡尔积。
- **数据丢失**：分区未加载、外部表误删 HDFS 数据、`OVERWRITE` 覆盖、路径权限问题。
- **元数据不一致**：HDFS 目录存在但 Metastore 无分区，或元数据存在但 HDFS 无数据，手动删文件导致。

---

## 二、综合示例（串联排查思路）

1. **执行慢排查**
    
    - 看执行计划是否走分区裁剪、谓词下推
    - 查看是否有 Reduce 倾斜，对倾斜 key 打散
    - 开启 MapJoin、合并小文件、调整执行引擎为 Spark
    
2. **OOM 排查**
    
    - 增大 `mapreduce.map.memory.mb` 和 reduce 内存
    - 减少并行度，防止并发过高
    - 检查是否存在不合理笛卡尔积
    - 拆分大 SQL，避免一次性 Shuffle 过多数据
    
3. **数据丢失排查**
    
    - 执行 `MSCK REPAIR TABLE` 恢复分区
    - 检查 HDFS 路径文件是否存在
    - 确认是否误执行 `DROP` 或 `OVERWRITE`
    - 核对同步任务是否成功
    
4. **元数据不一致排查**
    
    - `DESC FORMATTED` 对比 HDFS 路径与元数据
    - 手动添加缺失分区或删除无效分区
    - 避免直接在 HDFS 删文件，必须通过 DDL 操作
    

---

## 三、一句话总结

Hive 问题排查遵循**先看日志→查执行计划→定位倾斜 / 内存 / 分区问题**，执行慢优化逻辑，OOM 调资源，数据丢失修分区，元数据不一致做修复。
