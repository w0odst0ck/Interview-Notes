# 15. Hive 调度与任务依赖（配合 Airflow/DolphinScheduler/DataWorks）

## 一、简要面试介绍

- **任务调度**：定时自动执行 Hive SQL（ETL、分层同步、生成报表），替代手动跑数。
- **任务依赖**：保证执行顺序 —— 上游表生成完，下游才能跑，避免读脏数据、跑空数据。
- **常用调度工具**：
    
    - Airflow：Python 编写 DAG，灵活可控
    - DolphinScheduler：可视化拖拽，国产易用
    - DataWorks（阿里云）：一站式数据开发 + 调度 + 运维
    

## 二、综合示例（串联分层 + 依赖 + 调度）

典型数仓调度链路（按依赖顺序执行）：

1. 00:00 同步 MySQL → ODS 层（Sqoop/DataX）
2. ODS 完成后 → 跑 DWD 清洗明细
3. DWD 完成后 → 跑 DWS 轻度汇总
4. DWS 完成后 → 跑 ADS 业务指标
5. 全部成功后 → 触发报表 / 大屏刷新

在调度平台配置：

- 定时：每天 00:10 自动启动
- 依赖关系：严格上游成功才触发下游
- 失败策略：告警、重试、阻塞下游

## 三、一句话总结

Hive 任务通过 **Airflow/DolphinScheduler/DataWorks 实现定时调度**，按 **ODS→DWD→DWS→ADS 构建依赖链**，保证数仓自动化、有序、稳定运行。
