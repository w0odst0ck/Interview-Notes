# 一、简要面试介绍（只记关键点）

1. **分区裁剪**
    
    查询时先按分区字段过滤，只扫描对应分区目录，避免全表扫描。
    
2. **谓词下推（Predicate Pushdown）**
    
    把过滤条件`WHERE`尽量提前，在**数据读取阶段就过滤掉无效数据**，减少后续计算压力。
    
3. **MapJoin**
    
    大表关联小表时，将小表全量加载到内存，在 Map 阶段直接完成 Join，省去 Shuffle 过程，避免 Reduce 倾斜。
    
4. **并行执行**
    
    开启 Hive 并行参数，让**不相关的 stage / 任务同时执行**，提升整体任务速度。
    
5. **小文件处理**
    
    小文件过多会导致 NameNode 压力大、启动大量 Map 任务变慢，通过**合并输入 / 输出**解决。

# 二、综合示例（串联所有优化点）

```sql
SET hive.auto.convert.join = true;       -- 开启 MapJoin
SET hive.exec.parallel = true;           -- 开启并行执行
SET hive.merge.mapfiles = true;          -- 合并小文件
SET hive.merge.size.per.task = 256000000;

-- 1. 分区裁剪 + 谓词下推：先按dt分区过滤，再按page过滤
-- 2. MapJoin：大表 user_log 关联小表 user_info
-- 3. 并行执行：多阶段并发
-- 4. 结果自动合并小文件
INSERT OVERWRITE TABLE dwd.user_page_stat PARTITION(dt='2025-04-01')
SELECT 
    a.uid,
    COUNT(*) AS pv
FROM 
    ods.user_log a
JOIN 
    dim.user_info b  -- 小表自动走 MapJoin
ON a.uid = b.uid
WHERE 
    a.dt = '2025-04-01'  -- 分区裁剪
    AND a.page = 'home';  -- 谓词下推
GROUP BY a.uid;
```

# 三、一句话总结

Hive SQL 优化核心是**先分区裁剪 + 谓词下推减少数据量，用 MapJoin 避免 Shuffle，开启并行提升速度，合并小文件稳定性能**。
