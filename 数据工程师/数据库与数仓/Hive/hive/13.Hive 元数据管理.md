# 13. Hive 元数据管理（元数据查询、血缘分析、影响分析）

## 一、简要面试介绍

- **元数据**：库、表、字段、分区、存储路径、所有者、创建时间等结构信息，存在 MySQL/PostgreSQL。
- **元数据查询**：通过 `show`/`desc` 语句或直接查询 Metastore 库表，快速获取表结构与信息。
- **血缘分析（数据血缘）**：追踪字段从哪张上游表生成、经过哪些计算，定位数据来源。
- **影响分析**：查看某张表 / 字段被哪些下游任务依赖，判断修改是否会影响报表、应用。

---

## 二、综合示例（串联知识点）

1. **元数据查询**

```sql
-- 查看库
SHOW DATABASES;
-- 查看表
SHOW TABLES;
-- 查看表结构
DESC dwd.user_login;
-- 查看详细信息（存储格式、路径、分区）
DESC FORMATTED dwd.user_login;
```

2. **血缘分析**
    
    通过 Atlas 或内置血缘解析：
    
    `ads.user_report.dau` ← `dws.user_daily_stat.uid` ← `dwd.user_login_detail.uid` ← `ods.user_login.uid`
    
3. **影响分析**
    
    若要修改 `ods.user_login.uid` 字段类型，可查到下游依赖：
    
    `dwd.user_login_detail` → `dws.user_daily_stat` → `ads.user_report`
    
    需同步修改下游避免报错。
    

---

## 三、一句话总结

Hive 元数据管理核心是**查询表结构信息、通过血缘追溯数据来源、通过影响分析判断表变更风险**，保障数据中台稳定迭代。
