# 1）面试简要介绍

- **数据类型**
    
    - 基础：`TINYINT/SMALLINT/INT/BIGINT`、`FLOAT/DOUBLE`、`STRING/VARCHAR/CHAR`、`BOOLEAN`、`DATE/TIMESTAMP`
    - 复杂：`ARRAY`、`MAP`、`STRUCT`、`UNION`（数仓常用前三种）
    
- **存储格式**
    
    - **TextFile**：默认，文本格式，可读性强，**空间大、查询慢**，适合临时数据 / 小表
    - **ORC**：列式存储，带索引、支持 predicate pushdown，**压缩高、查询快**，Hive 生态最优
    - **Parquet**：列式存储，跨引擎友好（Spark/Presto/Impala），**压缩与性能均衡**
    
- **压缩算法**
    
    - 轻量：`Snappy`（解压快，CPU 友好，主流）
    - 高压缩：`ZSTD` / `Gzip`（空间更小，解压稍慢）
    - 可切片：`BZip2`、`ZSTD`、`Snappy`（部分场景），MR/Spark 可并行读取
    

# 2）综合示例（串联）

建一张用户行为表，按天分区，使用 **ORC + Snappy** 存储：

```sql
CREATE TABLE dwd.user_behavior (
    uid    BIGINT,
    event  STRING,
    info   MAP<STRING, STRING>,
    ts     TIMESTAMP
)
PARTITIONED BY (dt STRING)
STORED AS ORC
TBLPROPERTIES ("orc.compress"="SNAPPY");
```

- 用 `BIGINT/STRING/MAP/TIMESTAMP` 适配埋点数据结构
- 选用 **ORC 列式存储** 提升聚合查询速度
- 搭配 **Snappy 压缩** 平衡存储空间与计算性能
- 分区字段 `dt` 采用 `STRING` 方便按天管理数据

# 3）一句话总结

Hive 支持基础与复杂数据类型，生产优先使用 **ORC/Parquet 列式存储 + Snappy/ZSTD 压缩**，在空间、查询性能与引擎兼容性上取得最优平衡。
