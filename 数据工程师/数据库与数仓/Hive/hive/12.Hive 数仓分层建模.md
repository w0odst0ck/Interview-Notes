# 12. Hive 数仓分层建模（ODS/DWD/DWS/DWT/ADS）

## 一、简要面试介绍

- **ODS（原始数据层）**：直接同步业务库数据，结构与原系统一致，不做清洗，只做分区落地。
- **DWD（明细数据层）**：对 ODS 清洗、关联、脱敏，保留**最细粒度明细数据**，构建统一事实表。
- **DWS（汇总数据层）**：按主题轻度聚合，如按用户 / 天 / 设备汇总行为宽表，提高查询效率。
- **DWT（数据宽表层）**：全量累计型宽表，如用户全生命周期宽表，多用于标签与画像。
- **ADS（应用数据层）**：面向业务指标产出结果表，供报表、大屏、产品直接使用。

---

## 二、综合示例（串联分层）

sql

```
-- ODS：原始日志
CREATE EXTERNAL TABLE ods.user_login (...) PARTITIONED BY (dt);

-- DWD：清洗后明细
CREATE TABLE dwd.user_login_detail (...) PARTITIONED BY (dt);
INSERT OVERWRITE TABLE dwd.user_login_detail PARTITION(dt)
SELECT uid, ip, trim(status) FROM ods.user_login WHERE dt;

-- DWS：用户日度汇总
CREATE TABLE dws.user_daily_stat (...) PARTITIONED BY (dt);
INSERT OVERWRITE TABLE dws.user_daily_stat PARTITION(dt)
SELECT uid, COUNT(*) AS login_cnt FROM dwd.user_login_detail GROUP BY uid,dt;

-- DWT：用户累计宽表
CREATE TABLE dwt.user_total_info AS
SELECT uid, SUM(login_cnt) AS total_login FROM dws.user_daily_stat GROUP BY uid;

-- ADS：业务指标报表
CREATE TABLE ads.user_report AS
SELECT dt, COUNT(DISTINCT uid) AS dau FROM dws.user_daily_stat GROUP BY dt;
```

---

## 三、一句话总结

Hive 数仓按 **ODS 原始入仓→DWD 明细清洗→DWS 轻度汇总→DWT 全量宽表→ADS 业务指标** 分层，实现数据规范、复用与高效查询。
