## 一、面试简要介绍

1. **内部表（Managed Table）**
    
    Hive 托管表，删除表时**元数据 + HDFS 数据一并删除**，数据路径由 Hive 管理，适合数仓内部中间层表。
    
2. **外部表（External Table）**
    
    数据由用户 / 工具管理，删除表**只删元数据，不删 HDFS 原数据**，生产几乎都用外部表，安全可控。
    
3. **分区表（Partitioned Table）**
    
    按一个 / 多个字段（如 dt、city）将数据分到不同 HDFS 目录，查询时**分区裁剪**，大幅减少扫描数据量。
    
4. **分桶表（Bucketed Table）**
    
    对某列哈希后分到固定个数文件，用于**高效 Join、抽样**，优化大表关联性能。
    
5. **视图（View）**
    
    虚拟表，只存 SQL 定义不存数据，查询时动态解析，用于简化 SQL、权限屏蔽。
    
6. **物化视图（Materialized View）**
    
    物理存储预计算结果，支持自动刷新，用于**加速聚合查询**，数仓汇总层常用。
    

---

## 二、综合示例（串联）

```sql
-- 外部分区表（ODS层）
CREATE EXTERNAL TABLE ods.user_log (
    uid BIGINT, page STRING
) PARTITIONED BY (dt STRING)
STORED AS ORC;

-- 分桶表（DWD层，优化Join）
CREATE TABLE dwd.user_info (
    uid BIGINT, name STRING
) CLUSTERED BY (uid) INTO 32 BUCKETS;

-- 视图（简化查询）
CREATE VIEW v_user_daily AS
SELECT uid, COUNT(*) AS pv FROM ods.user_log GROUP BY uid;

-- 物化视图（预计算DWS汇总）
CREATE MATERIALIZED VIEW mv_user_stat
AS SELECT dt, COUNT(DISTINCT uid) AS uv FROM ods.user_log GROUP BY dt;
```

流程：外部分区表入湖 → 分桶表加速关联 → 视图简化口径 → 物化视图提升汇总查询速度。

---

## 三、一句话总结

生产优先用**外部表保证数据安全**，用**分区表提升查询效率**，**分桶表优化大表 JOIN**，**视图做逻辑封装、物化视图做预计算加速**。
