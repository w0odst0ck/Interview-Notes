## 一、简要面试版介绍

1. **Hive 架构与核心组件**
    
    Hive 是构建在 Hadoop 之上的数据仓库工具，把 SQL 翻译成 MapReduce/Spark/Tez 任务执行，核心组件如下：

- **Metastore（元数据存储）**：存储库名、表名、字段、分区、表类型、存储路径、字段类型等元信息，默认嵌 Derby，生产用 MySQL。
- **Driver（驱动器）**：接收 SQL，做**语法解析→语义解析→生成执行计划→优化计划**，统筹整个流程。
- **Execution Engine（执行引擎）**：执行最终的计算任务，支持 MR、Spark、Tez 等。
- **CLI/Beeline/JDBC**：提供用户交互入口。
- **HDFS**：实际数据存储层。

---

## 二、综合示例（串联所有知识点）

1. 你在终端用 Beeline 执行：

    ```sql
    SELECT user_id, COUNT(*) FROM dwd.user_login WHERE dt='2025-04-01' GROUP BY user_id;
    ```
    
2. **Driver** 接收 SQL，先解析语法、校验表 / 字段是否合法；
3. 去 **Metastore** 查询 `dwd.user_login` 是外部分区表、字段类型、分区字段 `dt`、数据在 HDFS 的路径；
4. Driver 生成逻辑执行计划并优化，提交给 **Execution Engine**（比如 Spark）；
5. 引擎去 **HDFS** 读取对应分区数据，执行计算并返回结果。

---

## 三、一句话总结

Hive 通过 Driver 解析优化 SQL，依托 Metastore 管理表结构元数据，借助 Execution Engine 执行分布式任务，最终读写 HDFS 数据实现类 SQL 数仓查询。
