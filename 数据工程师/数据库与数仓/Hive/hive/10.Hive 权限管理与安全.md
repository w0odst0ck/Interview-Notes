- （RBAC、行 / 列级权限、Kerberos）
## 一、简要面试介绍

1. **RBAC 权限模型**
    
    基于角色的权限控制：先创建**角色**，给角色赋库 / 表权限，再把角色授予用户，统一管理、便于维护。
    
2. **库 / 表级权限**
    
    常用权限：`SELECT、INSERT、UPDATE、DELETE、ALTER、DROP`
    
    控制谁能读写哪张表、哪个库。
    
3. **行级权限**
    
    通过**行过滤策略**，让不同用户只能看到符合条件的行（如只能看自己部门数据）。
    
4. **列级权限**
    
    控制用户只能查询指定列，对敏感列（手机号、身份证）做**脱敏或屏蔽**。
    
5. **Kerberos 安全认证**
    
    Hadoop 生态统一认证体系，实现**身份鉴权 + 加密传输**，防止非法访问，生产集群必开。

## 二、综合示例（串联）

```sql
-- 1. 创建角色并赋权限（RBAC）
CREATE ROLE analyst;
GRANT SELECT ON DATABASE dwd TO ROLE analyst;
GRANT ROLE analyst TO USER zhangsan;

-- 2. 列级权限：只允许查部分列
GRANT SELECT (uid, login_time) ON dwd.user_login TO ROLE analyst;

-- 3. 行级权限：只能看某城市数据
CREATE ROW POLICY user_policy ON dwd.user_login
FOR ROLE analyst
USING (city = 'beijing');

-- 4. Kerberos 环境下访问
kinit zhangsan
beeline -u "jdbc:hive2://xxx:10000/default;principal=hive/_HOST@EXAMPLE.COM"
```

## 三、一句话总结

Hive 采用 **RBAC 角色统一赋权**，配合**行 / 列级权限实现细粒度数据访问控制**，通过 **Kerberos 完成集群身份认证与安全加固**。
