## 一、简要面试介绍

**数据倾斜**：某一个 / 几个 Reduce 处理的数据远多于其他，导致任务卡住、跑不完。

常见倾斜场景：`group by`、`join`、`count(distinct)`、空值 / 特殊值过多。

### 定位方法

- 看执行日志：某 Reduce 耗时极长、处理行数远大于其他
- 看执行计划：stage 卡在 reduce
- 看 key 分布：某 key 数据量占比极高（空值、固定值、热点）

---

## 二、通用解决方案（必背）

1. **参数调优（group by 倾斜）**
    
    `set hive.groupby.skewindata=true;`
    
    开启两阶段聚合：先局部聚合，再全局聚合。
    
2. **空值 / 异常值打散（join/group 倾斜）**
    
    给倾斜 key 加随机后缀，分散到不同 Reduce。
    
3. **MapJoin 解决小表 join 大表倾斜**
    
    小表加载内存，避免 shuffle 倾斜。
    
4. **count (distinct) 优化**
    
    改用 `group by` + `count` 避免单 reduce 瓶颈。
    
5. **热点 key 单独处理**
    
    热点数据单独计算，最后 union 合并。
    

---

## 三、综合示例（串联）

```sql
-- 场景：user_log 中 uid=0 空值过多导致 join 倾斜
SET hive.auto.convert.join=true;
SET hive.groupby.skewindata=true;

INSERT OVERWRITE TABLE dwd.user_stat PARTITION(dt='2025-04-01')
SELECT
  -- 倾斜key打散：空值加随机数
  CASE WHEN uid IS NULL OR uid=0 
       THEN CONCAT('0_',rand()) 
       ELSE uid 
  END AS uid,
  COUNT(*) AS pv
FROM ods.user_log
WHERE dt='2025-04-01'
GROUP BY
  CASE WHEN uid IS NULL OR uid=0 
       THEN CONCAT('0_',rand()) 
       ELSE uid 
  END;
```

---

## 四、一句话总结

Hive 数据倾斜优先通过 ** 参数开启倾斜优化、打散倾斜 key、使用 MapJoin、替换 count (distinct)** 解决，核心思路是把集中的 key 分散到多个 Reduce 并行处理。
