# 过拟合 & 欠拟合（数据分析师面试极简版）

## 一、过拟合（Overfitting）

**定义**：模型**太死记训练数据**，泛化差，在训练集表现极好，测试集很差。

**怎么判断**：

- 训练集准确率 / 误差**很好**
- 验证 / 测试集**明显变差**
- 模型曲线**极度曲折**，把噪声也学进去

**怎么解决**：

1. 增加数据量
2. 正则化（L1、L2）
3. 降低模型复杂度（少特征、浅树）
4. 早停（Early Stopping）
5. dropout、数据增强

---

## 二、欠拟合（Underfitting）

**定义**：模型**太简单**，连训练数据规律都没学会，两边都差。

**怎么判断**：

- 训练集**表现就很差**
- 测试集也差
- 模型过于简单，拟合能力不足

**怎么解决**：

1. 增加特征、做特征工程
2. 提升模型复杂度（深树、更多参数）
3. 减少正则化
4. 训练更久

---

## 一句话记忆

- **过拟合**：学太细，**记住噪声** → 要**简化 / 正则**
- **欠拟合**：学太浅，**没学会规律** → 要**加复杂 / 加特征**
