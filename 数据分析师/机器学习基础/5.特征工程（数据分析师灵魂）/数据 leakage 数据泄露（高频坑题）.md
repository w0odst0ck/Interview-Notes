# 数据泄露 Data Leakage（面试**必考坑题**）

我给你整理成**一句话本质 + 常见场景 + 怎么避坑**，面试直接说。

## 一、一句话本质

**训练集用到了「只有测试集 / 未来才会有的信息」，导致线下分数虚高，上线崩掉。**

## 二、最常见 5 大泄露场景（背这些）

1. **归一化 / 标准化泄露（最常考）**
    
    - 错误：先对**全体数据**做归一化，再分 train/test
    - 正确：**先分 train/test，再用 train 的均值 / 方差 去标准化 test**
    
2. **特征构造泄露**
    
    - 用**全量数据**统计：全局均值、分位数、分桶边界
    - 正确：只能用**训练集**统计信息
    
3. **时间穿越泄露（时序必坑）**
    
    - 用**未来数据**预测过去
    - 例：用 “明天的股价” 预测 “今天的涨跌”
    
4. **测试集数据参与训练**
    
    - 做特征选择、填充缺失值时偷看 test
    - 正确：test 全程**黑盒**，不能碰
    
5. **Label 相关信息泄露**
    
    - 构造特征时直接 / 间接用到标签信息
    - 例：用 “是否逾期” 去做分组统计
    

## 三、一句话避坑原则

**所有与数据分布相关的操作，只能在训练集上计算，再无损应用到测试集。**

## 四、面试背诵版（绝杀）

**数据泄露是训练集非法使用测试集或未来信息，导致线下虚高、线上失效；最典型是归一化、特征统计、时序穿越，必须先拆分再预处理。**
