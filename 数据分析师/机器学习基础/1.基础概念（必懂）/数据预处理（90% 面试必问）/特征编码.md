# 特征编码（数据分析师面试极简速记）

## 一、是什么

把**文字 / 类别**转成**数字**，让模型能计算。

## 二、常用方法

1. **Label Encoding（标签编码）**
    
    - 类别 → 0,1,2…
    - 适用：**有序类别**（如：差 < 中 < 好、低 < 中 < 高）
    - 不适用：无序分类（如：北京 / 上海 / 广州）
2. **One‑Hot Encoding（独热编码）**
    
    - 一列变多列，只有 0/1
    - 适用：**无序分类**（性别、城市、品类）
    - 缺点：类别多会**维度爆炸**
3. **Ordinal Encoding（有序编码）**
    
    - 按业务顺序映射（如：小学 = 1, 中学 = 2, 大学 = 3）
    - 适用：**有大小 / 顺序关系**的特征
4. **Count / Target Encoding（计数 / 目标编码）**
    
    - 用频次或目标均值替换
    - 适用：**高基数特征**（如 ID、手机号、大量品类）

## 三、一句话口诀

- **有序用标签 / 顺序编码**
- **无序用独热**
- **类别太多用计数 / 目标编码**
